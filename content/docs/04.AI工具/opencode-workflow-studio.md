## 项目背景

随着 AI Agent 技术的发展，越来越多的开发者开始使用 AI 工具来辅助编程。然而，在实际使用中，我发现以下问题：

1. **单 Agent 局限性**：一个 Agent 很难同时处理复杂的多步骤任务
2. **上下文管理困难**：长时间对话容易丢失重要信息
3. **任务协作困难**：多个 Agent 之间难以协作
4. **日志追踪困难**：难以追踪 Agent 的执行过程

因此，我设想了一个基于 opencode 的可视化工作流平台。

## 核心功能

### 1. 可视化工作流编辑器

通过拖拽式界面，用户可以轻松创建工作流：

- **节点类型**：Agent 节点、条件节点、循环节点、并行节点
- **连接方式**：拖拽连线，支持分支和合并
- **参数配置**：每个节点可以配置输入输出参数

### 2. 多 Agent 管理

支持管理和调度多个 Agent：

- **Agent 池**：预定义多个 Agent，每个 Agent 有特定的能力
- **动态分配**：根据任务需求动态分配 Agent
- **负载均衡**：支持 Agent 的负载均衡

### 3. 任务流执行

支持复杂的任务流执行：

- **顺序执行**：按顺序执行多个任务
- **并行执行**：同时执行多个任务
- **条件分支**：根据条件选择执行路径
- **循环执行**：支持循环执行任务

### 4. 日志和监控

提供完善的日志和监控功能：

- **执行日志**：记录每个节点的执行日志
- **实时监控**：实时查看工作流执行状态
- **错误追踪**：追踪和定位错误

## 技术架构

### 前端技术栈

- **React**：前端框架
- **TypeScript**：类型安全
- **Ant Design**：UI 组件库
- **ReactFlow**：流程图编辑器
- **Zustand**：状态管理

### 后端技术栈

- **Node.js**：运行时环境
- **Express**：Web 框架
- **TypeScript**：类型安全
- **SQLite**：轻量级数据库
- **WebSocket**：实时通信

### 核心模块

1. **工作流引擎**：解析和执行工作流
2. **Agent 调度器**：管理和调度 Agent
3. **日志系统**：记录和查询日志
4. **配置管理**：管理系统配置
